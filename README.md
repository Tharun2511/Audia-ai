# Audia — AI Meeting Transcriber

Audia is an AI-powered meeting transcription app. Sign up, record a conversation in your browser (or upload an audio file), and get back a multi-speaker transcript with an AI summary you can chat with. Every session is private to your account and synced across devices.

## Features

### Capture & analysis
- **Browser recording** — one-click record via the MediaRecorder API, no installs needed
- **Audio file upload** — drag-and-drop any common audio format (mp3, wav, webm, m4a, ogg, flac)
- **Speaker diarization** — multiple speakers automatically identified as User1, User2, etc. with distinct colors
- **Accurate transcription** — powered by Deepgram nova-2 with smart formatting and punctuation
- **Timestamps** — each speaker segment shows its start time
- **AI summary** — 1–3 bullet TL;DR generated by Groq (LLaMA 3.1) immediately after transcription
- **Ask the transcript** — streaming chat against your session via Groq, with quick-prompt suggestions

### Editing & export
- **Speaker renaming** — rename `User1` → "Alice" inline; changes persist across devices
- **Title rename** — give each session a custom name
- **Copy to clipboard** — formatted transcript with speakers and timestamps
- **Delete** — remove individual sessions from your history

### Account & access
- **Email + password authentication** — signup, login, sign-out, with bcrypt-hashed passwords and JWT session cookies (HttpOnly, SameSite=Lax)
- **Per-user isolation** — every API endpoint verifies the session server-side; no cross-account access possible
- **Cross-device history** — sessions are tied to your account, not to a browser

### UI / UX
- **Hub-and-detail layout** — sidebar lists every session, main pane shows the one you've selected (or the recorder when starting fresh). One thing on screen at a time.
- **Dark mode** — light / dark / system, persisted in localStorage; switches instantly via CSS variables with no flash
- **Material UI v9 throughout** — themed components (TextFields, Buttons, Cards, Drawer, AppBar, Tooltips) tuned to the Audia brand
- **Toast notifications** — sonner-powered error feedback for any failed fetch
- **Mobile-friendly** — sidebar collapses into a drawer behind a hamburger on small screens
- **Microphone error guidance** — clear instructions if the browser blocks mic access

## Tech Stack

| Layer | Technology |

## API Endpoints

All endpoints **require an authenticated session**. Unauthenticated requests get `401 Unauthorized` with no body.

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/transcribe` | Receives audio (multipart), transcribes via Deepgram, generates summary via Groq, saves to the session user's history |
| `GET` | `/api/transcriptions` | Returns the last 20 sessions for the authenticated user |
| `PATCH` | `/api/transcriptions/:id` | Update title and/or rename speakers (only on sessions owned by the caller) |
| `POST` | `/api/transcriptions/:id/summary` | Regenerate the summary on demand |
| `DELETE` | `/api/transcriptions/:id` | Delete a session |
| `POST` | `/api/chat` | Groq streaming chat (LLaMA 3.1) — body: `{ prompt: string }`, response: `text/plain` stream |

Authentication is handled via **Server Actions**, not REST endpoints:

| Action | Where | What it does |
|---|---|---|
| `signup` | `src/app/actions/auth.ts` | Validate fields → hash password → insert user → create session → redirect to `/` |
| `login` | `src/app/actions/auth.ts` | Validate fields → bcrypt-compare → create session → redirect to `/` |
| `logout` | `src/app/actions/auth.ts` | Delete session cookie → redirect to `/login` |

### POST /api/transcribe

**Request:** `multipart/form-data` — single field `audio` (audio blob). The user is identified by the `session` cookie; no `email` field is sent or trusted from the client.

**Response:**
```json
{
  "id": "uuid",
  "duration": 42.3,
  "summary": "• Key topic one\n• Decision made\n• Action item",
  "segments": [
    { "speaker": "User1", "text": "Hello, how are you?", "start": 0.1, "end": 2.4 },
    { "speaker": "User2", "text": "I'm doing great, thanks.", "start": 2.8, "end": 5.1 }
  ]
}
```

### PATCH /api/transcriptions/:id

**Request body** (all fields optional):
```json
{
  "title": "Q2 Planning Call",
  "speakerMap": { "User1": "Alice", "User2": "Bob" }
}
```

The route verifies the session AND filters by `userEmail` so you can only patch your own sessions; anything else returns 404 (not 403 — we don't leak existence).

## Local Setup

### 1. Clone and install

```bash
git clone https://github.com/Tharun2511/Audia-ai.git
cd Audia-ai
npm install
```

### 2. Environment variables

Copy `.env.example` to `.env` and fill in your keys:

```bash
cp .env.example .env
```

```env
DATABSE_URL=postgresql://<user>:<password>@<host>/neondb?sslmode=require&channel_binding=require
DEEPGRAM_API_KEY=your_deepgram_api_key
GROQ_API_KEY=your_groq_api_key
SESSION_SECRET=<a 32-byte base64 string>
```

> **Note:** the database variable is `DATABSE_URL` (typo preserved from the original setup) — use exactly this spelling, or set `DATABASE_URL` instead; the data source reads either.

Generate a session secret:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

Get your keys:
- **Neon** — [neon.tech](https://neon.tech) — free tier, serverless PostgreSQL
- **Deepgram** — [deepgram.com](https://deepgram.com) — free $200 credit on signup
- **Groq** — [console.groq.com](https://console.groq.com) — free tier

### 3. Run

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). You'll be redirected to `/signup`. The database schema is auto-created on first run via TypeORM `synchronize: true` — no migrations needed.

### 4. Browser microphone permission

On first record, the browser will prompt for mic access — click **Allow**. If you previously blocked it:

> Chrome: click the 🔒 lock icon → Site settings → Microphone → set to **Ask** or **Allow** → refresh

## Deployment (Vercel)

1. Push to GitHub
2. Import the repo at [vercel.com/new](https://vercel.com/new)
3. Add environment variables in the Vercel dashboard:

| Key | Value |
|---|---|
| `DATABSE_URL` | Neon connection string |
| `DEEPGRAM_API_KEY` | Deepgram API key |
| `GROQ_API_KEY` | Groq API key |
| `SESSION_SECRET` | A **fresh** 32-byte base64 string (do not reuse the local one) |

4. Deploy — Vercel auto-detects Next.js, no build config needed.

> **Important:** if `SESSION_SECRET` is missing on Vercel, login will return a 500 with "An error occurred in the Server Components render." Check the Vercel function logs to confirm.

## How It Works

1. User visits the app → `proxy.ts` checks for a valid session cookie. If absent, redirect to `/login`.
2. User signs up or logs in via a Server Action. Password is bcrypt-hashed before insertion. On success, the server signs a JWT (`{ userId, expiresAt }`) and sets it as an HttpOnly `session` cookie, then redirects to `/`.
3. The home page is a server component — it calls `getCurrentUser()` from the DAL, which decodes the cookie and queries the user. The user's email and name are passed as plain props into `HomeClient`.
4. The browser shows the hub-and-detail UI: sidebar with the session list, main pane in the "Ready" state.
5. **To record:** click the big red record button → browser requests mic via `getUserMedia` → `MediaRecorder` captures audio in 250ms chunks → on **Stop**, chunks become a Blob and POST to `/api/transcribe`.
6. **To upload:** drop or pick an audio file in the upload zone — same endpoint.
7. The route handler `/api/transcribe` verifies the session, runs Deepgram (`nova-2` with `diarize: true`), groups consecutive words by speaker into segments, generates a 1–3 bullet summary via Groq, and saves the row to Postgres scoped to `userEmail` from the session.
8. The new session is auto-selected in the sidebar and rendered in the main pane (Summary + Transcript + Chat).
9. Renames, summary regenerations, deletes — all go through PATCH/POST/DELETE endpoints that filter by both `id` AND owner, so cross-account access is impossible.
10. Switching theme, signing out, navigating between sessions: all stay client-side until they need to.
