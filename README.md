# Audia — AI Meeting Transcriber

Audia is an AI-powered meeting transcription app. Sign up, record a conversation in your browser (or upload an audio file), and get back an accurate multi-speaker transcript with an AI summary. Then **chat with your transcript** (retrieval-augmented, with citations, memory, and tools), and **search semantically across every meeting** you've recorded. Every session is private to your account and synced across devices.

> Audia doubles as a from-scratch **AI-engineering curriculum** — RAG, evals, agents, MCP, and LangChain/LangGraph were each built primitives-first and documented in [`learning/`](learning/) (cheat sheets + a master revision doc).

## Features

### Capture & transcription
- **Browser recording** — one-click record via the MediaRecorder API, no installs
- **Audio file upload** — drag-and-drop common formats (mp3, wav, webm, m4a, ogg, flac)
- **Speaker diarization** — speakers auto-identified (User1, User2, …) with distinct colors, renameable inline
- **Accurate transcription** — Deepgram nova-2, smart formatting + punctuation + per-segment timestamps
- **AI summary** — 1–3 bullet TL;DR generated by Groq immediately after transcription

### AI: chat, retrieval & agents
- **Chat with your transcript (RAG)** — on save, each transcript is chunked, embedded (Gemini) and stored in **pgvector**; questions retrieve the most relevant chunks (MMR re-ranking + lost-in-the-middle reorder) and the model answers **grounded in those chunks**
- **Inline citations** — answers carry `[N]` markers rendered as clickable chips that **seek the audio** to that moment
- **Conversation memory** — follow-up questions work ("and what about Bob?"); a rolling buffer of prior turns is threaded into each request
- **Cross-meeting agent (tool use)** — the chat model can call tools (`listMyMeetings`, `getMeetingDetails`) in an agentic loop to answer questions that span your whole library, not just the open session
- **Semantic + hybrid search across all meetings** — sidebar search: type to filter the session list instantly, press **Enter** for semantic search over transcript *content*. Combines dense (pgvector) + lexical (Postgres full-text) recall fused with **Reciprocal Rank Fusion**, then **cross-encoder re-ranking** (Jina); results show the meeting, timestamp, and a relevance score
- **MCP server** — `/api/mcp` exposes Audia's tools over the **Model Context Protocol** (JSON-RPC 2.0), so external MCP clients can query your meetings
- **LangGraph agent (parallel build)** — the chat agent is *also* implemented as a LangGraph `StateGraph` at `/api/chat-graph`, with per-thread memory (checkpointer) and a human-in-the-loop interrupt — alongside the hand-rolled version, for comparison
- **Eval harness** — golden sets + LLM-as-judge for the summarizer and the RAG chat, gated in CI (`npm run eval:all`)

### Editing & export
- **Speaker renaming** — rename `User1` → "Alice" inline; persists across devices (and re-chunks/re-embeds)
- **Segment text editing** + **title rename**
- **Copy / Download** — formatted transcript as `.txt` or `.md`
- **Delete with undo** — 5-second window before the server-side delete fires

### Account & access
- **Email + password auth** — bcrypt-hashed passwords, JWT session cookies (HttpOnly, SameSite=Lax)
- **Per-user isolation** — every endpoint verifies the session server-side and filters by owner; no cross-account access (vector + lexical search are ownership-scoped in SQL)
- **Cross-device history** — sessions are tied to your account, not a browser

### UI / UX
- **Hub-and-detail layout** — sidebar lists sessions; the main pane is state-driven (Ready / Recording / Processing / Session / **Search results**)
- **Dark mode** — light / dark / system, persisted, no flash (CSS variables)
- **Material UI v9 throughout**, **sonner** toasts, **mobile drawer**, mic-permission guidance

## API Endpoints

All endpoints **require an authenticated session** (401 otherwise). Ownership is enforced in SQL — you only ever see your own data.

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/transcribe` | Audio (multipart) → Deepgram → summary → chunk + embed (pgvector) → save |
| `GET` | `/api/transcriptions` | List the user's sessions (newest first) |
| `PATCH` / `DELETE` | `/api/transcriptions/:id` | Rename / edit segments / delete (re-chunks on segment edits) |
| `GET` | `/api/transcriptions/:id/audio-url` | Short-lived signed playback URL |
| `POST` | `/api/chat` | RAG chat — `{ question, transcriptionId?, sessionId? }` → `text/plain` stream + `X-Citations` / `X-Chat-Session` headers. Retrieves chunks, runs the agentic tool loop, answers grounded with citations + memory |
| `GET` | `/api/search` | Hybrid semantic search across all meetings — `?q=&k=&n=` → ranked hits `{ meetingTitle, startTime, text, relevanceScore, sources }` |
| `POST` | `/api/mcp` | MCP server (JSON-RPC 2.0): `initialize`, `tools/list`, `tools/call` |
| `POST` | `/api/chat-graph` | LangGraph chat agent — `{ question, mode?, sessionId? }` → JSON. A/B against `/api/chat` |

Auth is handled via **Server Actions** (`signup` / `login` / `logout` in `src/app/actions/auth.ts`), not REST.

## Local Setup

```bash
git clone https://github.com/Tharun2511/Audia-ai.git
cd Audia-ai
npm install
cp .env.example .env   # then fill in keys
```

```env
DATABASE_URL=postgresql://<user>:<password>@<host>/neondb?sslmode=require
DEEPGRAM_API_KEY=your_deepgram_api_key      # transcription
GROQ_API_KEY=your_groq_api_key              # summary + chat/agent
GEMINI_API_KEY=your_gemini_api_key          # embeddings (RAG + search)
JINA_API_KEY=your_jina_api_key              # cross-encoder re-ranking (search). If unset, search falls back to bi-encoder order
SESSION_SECRET=<a 32-byte base64 string>
BLOB_READ_WRITE_TOKEN=<Vercel Blob token>   # audio persistence + playback (optional)
```

Generate a session secret: `node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"`

Get keys: **Neon** ([neon.tech](https://neon.tech)) · **Deepgram** ([deepgram.com](https://deepgram.com)) · **Groq** ([console.groq.com](https://console.groq.com)) · **Gemini** ([aistudio.google.com](https://aistudio.google.com)) · **Jina** ([jina.ai/reranker](https://jina.ai/reranker/)).

```bash
npm run dev          # http://localhost:3000 — schema auto-created via TypeORM synchronize
```

> The pgvector extension + full-text bits are set up automatically on first DB connection.

## Deployment (Vercel)

Push to GitHub → import at [vercel.com/new](https://vercel.com/new) → add the env vars above (use a **fresh** `SESSION_SECRET`; `BLOB_READ_WRITE_TOKEN` is auto-injected when you connect a Vercel Blob store) → deploy. Vercel auto-detects Next.js.

> If `SESSION_SECRET` is missing on Vercel, login returns a 500 — check the function logs.

## How It Works

1. `proxy.ts` gates routes on the session cookie; unauthenticated users go to `/login`.
2. Signup/login via Server Actions → bcrypt hash → signed JWT in an HttpOnly cookie.
3. **Record/upload** → `/api/transcribe` runs Deepgram (`nova-2`, diarized), groups words into speaker segments, summarizes via Groq, then **chunks the transcript, embeds each chunk (Gemini) and stores it in pgvector** — all scoped to the session user.
4. **Chat** → `/api/chat` embeds the question, retrieves top-k chunks from pgvector (MMR + edge-reorder), threads in recent conversation turns, and runs an agentic loop (the model may call tools across your meetings) — streaming a grounded answer with `[N]` citations that seek the audio.
5. **Search** → `/api/search` runs dense (pgvector) + lexical (Postgres FTS) recall in parallel, fuses them with RRF, re-ranks with a cross-encoder, and returns ranked moments across every meeting.
6. **Interop** → external MCP clients hit `/api/mcp`; the LangGraph build at `/api/chat-graph` mirrors the chat agent with checkpointer memory + interrupts.
7. Every read/write filters by both `id` and owner, so cross-account access is impossible.
