# Accessibility — non-negotiables

Target: WCAG 2.2 AA. Everything below is enforced, not aspirational.

## Color & contrast

- Body text contrast ≥ 4.5:1 against `background.default` AND `background.paper`. The current palette is tested for this. If you change a hex, re-test.
- Don't encode state by color alone. Pair color with an icon, label, or shape.
  - ✅ Error chip = red icon + "Error" label.
  - ❌ Just a red dot.
- Focus ring always visible on `:focus-visible`. Don't `outline: none` without replacement.

## Focus

- MUI default focus rings are fine; don't disable them.
- Custom interactive elements (a `<div onClick>`) must be a `<Button>`, `<IconButton>`, `<MenuItem>`, or wrapped in `ButtonBase`.
- Tab order follows visual order. If you need to control it, use `tabIndex={0}` sparingly — never `tabIndex` > 0.
- Modals: focus traps automatically via MUI `Dialog` / `Drawer`. Don't break it (don't `disableEnforceFocus` unless you really know why).

## Keyboard

| Action | Required keys |
|---|---|
| Submit form | Enter |
| Cancel / Close dialog | Esc |
| Toggle menu | Enter / Space |
| Navigate list | Up / Down arrows (use `MenuList` or arrow handlers on `List`) |
| Future: command palette | ⌘K / Ctrl+K |

Every action that has a mouse path must have a keyboard path. Test by unplugging the mouse for a session.

## ARIA

- `aria-label` on all icon-only buttons. Don't rely on Tooltip alone — Tooltip text is announced after a delay.
- `aria-live="polite"` on regions that update with new content (live transcript, chat streaming response).
  - Use `aria-atomic="false"` so only the new piece is announced, not the whole region.
- `role="status"` on inline loading messages.
- `aria-busy="true"` on a region during fetch.
- Don't add ARIA where semantics suffice — `<button>` already announces correctly.

## Live regions (for streaming content)

For live transcription:

```tsx
<Box aria-live="polite" aria-atomic="false" sx={{ /* visual styles */ }}>
  {finalSegments.map((s) => <Caption key={s.id} text={s.text} />)}
</Box>

{/* interim segments shown visually but NOT announced */}
<Box aria-hidden="true" sx={{ fontStyle: "italic", color: "text.disabled" }}>
  {interimText}
</Box>
```

Announce **final** segments only, not interim/partial ones. Otherwise screen-readers stutter.

For streaming chat responses:

```tsx
<Box aria-live="polite" aria-atomic="false">
  {streamedTokensSoFar}
</Box>
```

Once streaming ends, the final message is naturally announced as a whole.

## Forms

- Every input has a visible `<label>` (use `TextField`'s `label` prop, not placeholder-as-label).
- Errors: `helperText` + `aria-invalid={true}` + a sonner toast on submit failure for sighted users.
- Submit buttons: `disabled` during pending action. Use `aria-busy` if there's text inside.
- Password fields: use the `PasswordInput` component — already wired with proper `autoComplete` and toggle.
- `<form noValidate>` — Zod validates, browser doesn't. But still set semantic `type="email"`, `type="password"`, `autoComplete` correctly so password managers work.

## Images / icons

- Decorative icons inside a `Button` with a text label: no `aria-hidden` needed; MUI handles it.
- Standalone meaningful icons (like a record dot): add `aria-label` to the wrapper.
- Avatar with initials: `alt={user.name}` (initials alone aren't enough).
- Brand logo SVG: `aria-label="Audia"` on the wrapping link/button.

## Motion

Respect `prefers-reduced-motion`. See [motion.md](motion.md).

## Toasts (sonner)

Sonner toasts are announced via ARIA live regions automatically. But:
- Don't put critical information in a toast (it auto-dismisses).
- For "you must do X to continue" — use a `Dialog`, not a toast.
- Don't stack > 3 toasts at once — they queue, but visually it's noise. Group related events into one toast with a `description`.

## Time-sensitive UI

For session timers (recording timer):
- Show the value visually, but only announce changes every minute via `aria-live="polite"` (not every second).
- Use `role="timer"` on the container with a hidden update region:

```tsx
<Box role="timer" aria-live="polite" aria-atomic="true">
  <span>{visibleSeconds}</span>
  <Box sx={{ position: "absolute", left: -9999 }}>
    {minuteOnlyAnnouncement}
  </Box>
</Box>
```

## Testing checklist (quick — before "done")

1. Tab through the surface. Does focus go where you expect? Is the ring visible on every interactive element?
2. Run a Lighthouse a11y audit on the page (`npm run dev` → DevTools → Lighthouse). Target ≥ 95.
3. Toggle prefers-reduced-motion in DevTools → Rendering → Emulate. Confirm no animations run that shouldn't.
4. Zoom to 200% (Ctrl+`+` ×4). Layout still works, no clipped controls.
5. (Stretch) VoiceOver / Narrator on the critical flow (record → see summary).

## Common pitfalls

- Using a `<div onClick>` instead of `<Button>` — div has no role, no keyboard handler, no focus ring.
- Using `placeholder` as the only label — placeholder disappears on focus.
- `aria-live="assertive"` on chat streaming — too aggressive; use `"polite"`.
- Removing focus styles to "clean up the look" — you've just made the app unusable by keyboard users.
- Color-only error states — pair with `<ErrorOutlineIcon />` and an explicit message.
